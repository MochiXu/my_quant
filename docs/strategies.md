# 交易策略设计

> 配套文档，对应 `docs/design.md` 的「信号核」抽象。本文回答：真实交易里大家用什么
> 策略、成交量数据怎么用、宏观（央行/美联储）因素怎么用，以及针对纳指 / 沪深 300
> ETF 的具体策略思路。

## 0. 一个前提认知：策略不是「一招」，是「分层堆叠」

新手常以为存在某个「最强策略」。真实做法是把几层独立的东西叠起来：

```
   ┌─────────────────────────────────────────────┐
   │ 第4层  组合构建  多个信号 → 一组目标权重        │
   ├─────────────────────────────────────────────┤
   │ 第3层  仓位/风控  这次该下多重？(波动率目标…)   │
   ├─────────────────────────────────────────────┤
   │ 第2层  环境过滤  现在该不该做？(宏观/波动/量能) │
   ├─────────────────────────────────────────────┤
   │ 第1层  alpha 信号  方向从哪来？(趋势/回归/估值) │
   └─────────────────────────────────────────────┘
```

- **第 1 层 alpha 信号**：判断方向（涨/跌、买哪只）。趋势、均值回归、估值、套利。
- **第 2 层 环境过滤**：判断「现在适不适合用这个信号」。宏观周期、波动率高低、量能配合。
- **第 3 层 仓位与风控**：决定「下多重」。波动率目标、风险平价、止损。
- **第 4 层 组合构建**：把多个标的、多个信号合成最终 `{标的: 权重}`。

你现在的双均线只覆盖了第 1 层的一种、动量轮动覆盖了第 1+4 层的一部分——所以显得单薄。
真实策略的「厚度」来自把 2、3 层补上，而不是去找更花哨的第 1 层信号。

> 对应到框架：第 1 层是 `Strategy` 信号核；第 2、3 层可以做成**可组合的 overlay**
> （包在信号核外面、对权重做缩放/门控的装饰器）；第 4 层是 `core/portfolio.py`。

## 1. 第 1 层：alpha 信号族

适合你这种「日级、一天 1~2 次、ETF」的策略族（剔除了需要日内/低延迟的高频类）：

### 1.1 趋势 / 动量（Trend / Momentum）

| | |
|---|---|
| 原理 | 涨的继续涨、跌的继续跌；顺势持有，破势离场。 |
| 为什么有效 | 行为金融：投资者对信息**反应不足**，趋势缓慢扩散；学术支撑充分（Jegadeesh & Titman 1993；Moskowitz/Ooi/Pedersen 2012 时序动量）。 |
| 适用 | 中长周期（数周~数月），几乎所有可交易品种。**最适合 ETF 指数**。 |
| 数据 | 只要 OHLCV。 |
| 典型实现 | 双均线、MACD、价格 vs 200 日均线、唐奇安通道突破（Donchian breakout）、N 月收益排名。 |
| 主要陷阱 | 震荡市来回打脸（whipsaw）；参数容易过拟合——务必 walk-forward。 |

- **时序动量**：单标的，看自己过去的走势（你的双均线属于此类）。
- **截面动量**：多标的，看相对排名（你的动量轮动属于此类）。

### 1.2 均值回归（Mean Reversion）

| | |
|---|---|
| 原理 | 价格偏离「中枢」太远会被拉回；超跌买、超涨卖。 |
| 为什么有效 | 短周期上投资者**反应过度** + 流动性提供者赚回归的钱。 |
| 适用 | **短周期**（几天~两周）。注意：和趋势在不同时间尺度上同时成立——短期回归、长期趋势。 |
| 数据 | OHLCV；配对交易还要两个标的的价差。 |
| 典型实现 | RSI 超买超卖、布林带、价格对均线的 z-score、配对交易 / 统计套利（如沪深 300 ETF vs 中证 500 ETF 的价差回归）。 |
| 主要陷阱 | 「接飞刀」——真正的趋势反转会让回归策略持续亏损；必须配硬止损或趋势过滤。 |

### 1.3 估值 / 基本面（Valuation）

| | |
|---|---|
| 原理 | 指数便宜时多配、贵时减配。 |
| 为什么有效 | 估值多年期均值回归；A 股宽基指数（沪深 300）的 PE/PB 历史分位是国内很流行的「估值温度计」。 |
| 适用 | **慢变量**，决定的是「战略仓位中枢」，不是日内择时。最适合宽基 ETF。 |
| 数据 | 指数 PE / PB / 股息率历史（tushare `index_dailybasic`、akshare 指数估值接口）；股权风险溢价 ERP = 盈利收益率 − 国债收益率。 |
| 典型实现 | PE 百分位分档：<20% 分位重仓、>80% 分位轻仓，中间线性过渡。 |
| 主要陷阱 | 分位会随指数成分/盈利结构漂移（结构性变化）；「便宜还能更便宜」，单独用择时很差，应与趋势结合。 |

### 1.4 套息 / 持有收益（Carry）

| | |
|---|---|
| 原理 | 持有本身就有正收益的资产（股息、债息、货币基金利息）在「没有明确信号」时是更好的默认仓位。 |
| 适用 | 主要作为**现金的替代**：空仓时不躺现金，放红利 ETF / 国债 ETF / 货币 ETF。 |
| 数据 | 股息率、票息。 |

> 你的 quant_learn 标的池里已经有黄金 ETF、国债 ETF、银华日利（货币）——它们天然就是
> 轮动策略里「risk-off 时的避风港」。

## 2. 成交量 / 量价数据怎么用（回答你的问题二）

**结论先行**：成交量在日级 ETF 上，主要价值是**确认与过滤（第 2 层）**，很少能单独当
alpha（第 1 层）。它不告诉你方向，它告诉你「这个方向的信号可不可信」。

常用量价指标：

| 指标 | 含义 | 怎么用 |
|---|---|---|
| 成交量 / 成交额 | 当日交易活跃度 | 突破时放量 → 信号可信；缩量突破 → 大概率假突破。 |
| 量比 | 当日量 vs 近 N 日均量 | 衡量「相对放量」程度。 |
| 换手率 | 成交量 / 流通份额 | ETF 上反映场内热度。 |
| OBV（能量潮） | 按涨跌方向累计成交量 | 价涨量增/价跌量缩 = 健康；量价背离 = 警惕。 |
| VWAP | 成交量加权均价 | 多用于日内执行基准，日级策略用处小。 |
| 两市成交额 | 全市场量能 | **市场级情绪指标**：地量见底、天量见顶的经验规律。 |

务实建议：

- ETF 指数的量价关系**比个股弱**——指数没有「主力建仓」那种结构。
- 最实用的两个用法：① **放量过滤**——突破/金叉时要求成交量 > 近 20 日均量，过滤假信号；
  ② **全市场量能当情绪温度计**——两市成交额的极值区间。
- 不要堆一堆量价指标当独立因子，过拟合风险极高。

## 3. 宏观因素怎么用（回答你的问题三）

**结论先行**：央行/美联储等宏观因素**确实影响巨大**，但它是**慢变量**——适合做第 2 层
的「环境过滤 / 制度判断（regime）」，**不适合**拿来做日度快信号。原因：宏观数据低频、
有滞后、易被事后解读、样本少（几十年也就几个周期）→ 极易过拟合。

### 3.1 对你两个标的，宏观分别看什么

**纳指 ETF**（513100 等）——本质是「美股 + 汇率 + QDII 溢价」：

- 美联储政策：联邦基金利率、点阵图、FOMC 表态（加息周期压估值，降息周期利成长股）。
- 美债 10 年收益率：是纳指（高久期成长股）的「贴现率」，收益率快速上行 → 纳指承压。
- 美元指数 DXY、美国 CPI / 非农就业。
- VIX（恐慌指数）：飙升 = risk-off，可作为减仓过滤。
- **QDII 溢价**：场内价 vs 基金净值的溢价。国内额度紧张时溢价走高，溢价过高=场内过热，
  既是追高成本警告，也可做反向参考。

**沪深 300 ETF**（510300 等）——看国内：

- 中国央行：降准（RRR）、降息（MLF / LPR）——宽松周期利好权益。
- 社融、M2、PMI：经济动能的先行/同步指标。
- 北向资金流向：常被当「聪明钱」代理变量（注意：其预测力近年减弱，争议较大，当辅助）。

### 3.2 把宏观结构化：美林时钟（Investment Clock）

经典的宏观 regime 框架，用「经济增长」×「通胀」两维分四象限，指导大类资产配置：

| 象限 | 增长 / 通胀 | 占优资产 |
|---|---|---|
| 复苏 | 增长↑ 通胀↓ | 股票 |
| 过热 | 增长↑ 通胀↑ | 商品（黄金等） |
| 滞胀 | 增长↓ 通胀↑ | 现金 |
| 衰退 | 增长↓ 通胀↓ | 债券 |

它的价值不是精确择时，而是给「该偏向哪类资产」一个有逻辑的先验，正好对接你的多资产
轮动（纳指 / 沪深 300 / 黄金 / 债 / 货币）。

### 3.3 务实用法

- 宏观当 **regime 开关 / 权重缩放**：例如「美债收益率单月上行 > X bp → 纳指目标权重 ×0.5」。
- 不要把月度宏观数据塞进日度信号里直接决策——频率不匹配。
- 宏观因子数量要克制；每个都要能讲清经济学逻辑，否则就是数据挖掘。

## 4. 第 3 层：仓位与风控（最被低估、性价比最高）

很多人花全部精力找第 1 层信号，其实把这一层做好对夏普/回撤的提升常常更大：

| 方法 | 做什么 |
|---|---|
| 波动率目标（vol targeting） | 按近期波动率反向缩放仓位——波动大就减仓，让组合波动稳定。简单、稳健、显著降回撤。 |
| 风险平价（risk parity） | 多资产间按「风险贡献相等」分配权重（桥水全天候思路）。 |
| 固定比例 / 凯利分数 | 每次只投入资金的固定比例。 |
| 止损 / 移动止损 | 单笔亏损上限；跟踪止损锁利润。 |
| 无交易带 | 目标与现状偏离小于阈值不动手——降换手、省成本（design.md 已含）。 |

**强烈建议**：把「波动率目标」作为默认的第 3 层 overlay 加在所有策略上——它几乎不增加
过拟合风险，却能稳定改善风险调整后收益。

## 5. 针对你标的的三个具体策略思路（可直接进回测）

都遵循 design.md 的信号核契约（收盘决策、次日成交、输出目标权重）。

### 思路 A — 纳指 ETF 趋势 + 风险过滤

- 第 1 层：价格在 200 日均线上方 → 倾向持有（趋势）。
- 第 2 层：VIX 高于阈值 或 美债收益率快速上行 → 权重打折（risk-off 过滤）。
- 第 3 层：波动率目标缩放。
- 提示：QDII 溢价过高时别追，纳入参考价/择时。

### 思路 B — 沪深 300 ETF 估值分档 + 趋势确认

- 第 1 层：PE 历史百分位决定中枢权重（低估高配、高估低配）。
- 第 2 层：价格站上中长期均线才执行「加仓」——避免在便宜区一路下跌中接刀。
- 第 3 层：波动率目标缩放。

### 思路 C — 多资产双动量轮动（Dual Momentum）

Gary Antonacci 的经典框架，最契合你的轮动需求：

- **相对动量**：在 纳指 ETF / 沪深 300 ETF /（可加中证 500、黄金）里，选过去
  12 个月收益最高的。
- **绝对动量**：如果这个「赢家」的过去 12 月收益还跑不赢货币 ETF / 国债 ETF，
  就空仓避险（切到债/货币），而不是硬持有。
- 月度或双周再平衡，换手低、逻辑清晰、适合你 1~2 次/天的节奏。

## 6. 这些策略对框架的影响

当前 design.md 里信号核的输入 `panel` 只有 OHLCV。要支持量价/估值/宏观，需要把它扩成
一个**特征库（feature store）**：

| 数据类别 | 用于 | 来源 |
|---|---|---|
| OHLCV、成交量、换手率 | 趋势、量价过滤 | tushare / akshare（已有） |
| 指数 PE / PB / 股息率 | 估值择时 | tushare `index_dailybasic`、akshare 指数估值接口 |
| 两市成交额、北向资金 | 市场情绪 | akshare |
| 美债收益率、DXY、VIX、纳指现货 | 纳指 ETF 宏观过滤 | akshare / OpenBB / yfinance |
| LPR、MLF、社融、M2、PMI、CPI | 美林时钟 / 国内 regime | akshare 宏观接口 |

> 这是对 design.md 数据层的一处明确扩展。确认本文后，我会把 design.md 的 `data` 模块
> 从「只存 OHLCV」改为「分类特征库」，并把第 2/3 层 overlay 的抽象补进 `core/`。

## 7. 务实建议与陷阱

**从哪开始**（别一上来就堆复杂策略）：

1. 先把「趋势 + 波动率目标」这条最稳健的链路跑通、回测、walk-forward 验证。
2. 再加估值分档、宏观 regime 过滤，每加一层都单独验证它是否真的有增量。
3. 最后才碰多信号合成。

**反复强调的陷阱**：

- **过拟合**：因子越多、参数越多，越容易在历史上「完美」、在实盘上失效。每个信号都要
  问「经济学逻辑是什么」，问不出来就别用。你已有的 walk-forward 是核心防线。
- **未来函数**：宏观/财报数据有「发布滞后」，回测时必须用「当时真正能看到的值」。
- **频率不匹配**：月度宏观数据不要做日度决策。
- **交易成本**：高换手策略在回测里漂亮、扣掉佣金/滑点后归零——成本模型要诚实。
- **制度变化**：一个策略在某个市场周期有效不代表永远有效，要持续监控。
- **幸存者偏差 / 数据窥探**：别用「现在的热门 ETF 列表」回测十年前。

**一句话总结**：真实交易的「厚度」不在于找到更神的第 1 层信号，而在于把环境过滤、
仓位风控、组合构建这几层老老实实补齐，并用严格的样本外验证压住过拟合。
